# Пример результата логик-аудита

Ниже — результат проверки файла `sample-input.md`:

```
═══════════════════════════════════════════════
  ЛОГИК-АУДИТ РАССУЖДЕНИЙ
═══════════════════════════════════════════════

КАРТА ПОНЯТИЙ
┌──────────────────────────────────────────────┐
│ Термин: «цифровая трансформация»             │
│ Содержание (Dfn): «процесс цифровизации      │
│   бизнес-процессов с целью трансформации»    │
│ Объём: не определён                          │
│ Статус: КРУГ В ОПРЕДЕЛЕНИИ                   │
├──────────────────────────────────────────────┤
│ Термин: «эффективный сотрудник»              │
│ Содержание (Dfn): «сотрудник, который        │
│   эффективно выполняет обязанности»          │
│ Объём: все сотрудники? непонятно             │
│ Статус: КРУГ В ОПРЕДЕЛЕНИИ                   │
├──────────────────────────────────────────────┤
│ Термин: «риски»                              │
│ Содержание (Dfn): не определено явно         │
│ Объём: 4 категории (неполный)                │
│ Статус: НЕПОЛНАЯ КЛАССИФИКАЦИЯ               │
└──────────────────────────────────────────────┘

НАЙДЕННЫЕ ПРОБЛЕМЫ

[1] Круг в определении (circulus in definiendo)
    Тип: определение
    Цитата: «Цифровая трансформация — это процесс
             цифровизации бизнес-процессов компании
             с целью их трансформации»
    Формула: ⌜цифровая трансформация⌝ =Df
             ⌜цифровизация⌝ + ⌜трансформация⌝
    Dfd входит в Dfn: «трансформация» ∈ Dfn
    Нарушено правило: Dfd ∉ Dfn
    Рекомендация: определить через конкретные
    результаты — «переход от ручных процессов к
    автоматизированным с измеримыми KPI»

[2] Круг в определении (circulus in definiendo)
    Тип: определение
    Цитата: «Эффективный сотрудник — это сотрудник,
             который эффективно выполняет обязанности»
    Формула: ⌜эффективный⌝ =Df ... + ⌜эффективно⌝
    Dfd входит в Dfn
    Нарушено правило: Dfd ∉ Dfn
    Рекомендация: определить через метрики —
    «сотрудник, выполняющий ≥90% задач в срок
    с качеством ≥ установленного стандарта»

[3] Нарушение закона непротиворечия (contradictio)
    Тип: логический закон
    Цитата §2: «Все сотрудники поддерживают изменения»
    Цитата §4: «Не все сотрудники поддерживают
               изменения. Сопротивление персонала
               является ключевой проблемой»
    Формула: (∀x: P(x)) ∧ ¬(∀x: P(x)) — противоречие
    Нарушен закон: ¬(A ∧ ¬A)
    Рекомендация: убрать утверждение из §2 или
    уточнить — «по результатам опроса, 70%
    сотрудников поддерживают изменения»

[4] Смешение оснований классификации
    Тип: классификация
    Цитата: «Человеческие (молодые сотрудники
             не справляются)»
    Формула: основание₁ = тип риска (техн/фин/орг),
             основание₂ = возраст сотрудника
    Нарушено правило последовательности:
    1 деление = 1 основание
    Рекомендация: «Человеческие (сопротивление
    персонала, нехватка компетенций)» — без
    привязки к возрасту

[5] Неполная классификация (divisio insufficiens)
    Тип: классификация
    Цитата: «Риски делятся на: технические,
             финансовые, человеческие, организационные»
    Формула: X₁ + X₂ + X₃ + X₄ ≠ U
    Пропущены: регуляторные, стратегические,
    информационной безопасности
    Нарушена соразмерность: ∑Xᵢ < U(род)
    Рекомендация: дополнить или оговорить —
    «рассматриваем 4 ключевых категории из N»

[6] Модальная ошибка (◇ → □)
    Тип: аргументация
    Цитата: «Конкурент может внедрить аналогичную
             систему, поэтому мы обязаны ускорить
             внедрение в два раза»
    Формула: ◇A → □B
    Из возможности (◇) выведена необходимость (□)
    Рекомендация: «Если конкурент внедрит систему
    (◇A), это создаст риск отставания (◇B).
    Целесообразно рассмотреть ускорение» — без
    категоричного «обязаны»

[7] Нарушение закона достаточного основания
    Тип: аргументация
    Цитата: «Продажи упали на 3%. Следовательно,
             нужно полностью пересмотреть стратегию»
    Формула: A → B, но ¬∃C: (A ∧ C) → B
    Нет промежуточных посылок: почему 3% = полный
    пересмотр? Каков порог? Какие альтернативы?
    Рекомендация: добавить анализ причин падения
    и обосновать масштаб реакции

[8] Ложная дилемма (falsum dilemma)
    Тип: аргументация
    Цитата: «Либо мы трансформируемся, либо
             закрываемся»
    Формула: A ⊻ B, но ∃C, D, E...
    Представлены 2 варианта как исчерпывающие,
    но существуют: частичная трансформация,
    партнёрство, аутсорсинг, поэтапный переход
    Рекомендация: «Трансформация — приоритетный
    путь. Альтернативы: ...» — без ложной дихотомии

ИТОГ
├── Понятия: 3 проверено, 2 с проблемами
├── Определения: 2 проверено, 2 с кругами
├── Классификации: 1 проверено, 1 с проблемами
├── Логические законы: 1 нарушение (непротиворечие)
├── Аргументация: 3 проблемы
└── Общая оценка: содержит критические ошибки

РЕКОМЕНДАЦИИ ПО УСИЛЕНИЮ
1. Переписать определения — убрать круги, ввести
   измеримые критерии
2. Устранить противоречие §2/§4 — привести данные
   опросов или исследований
3. Дополнить классификацию рисков или явно
   ограничить её scope
4. Смягчить модальности — не выводить «обязаны»
   из «может быть»
5. Обосновать связь «падение 3% → пересмотр
   стратегии» промежуточными посылками
6. Убрать ложную дилемму в выводах — показать
   спектр альтернатив
```
